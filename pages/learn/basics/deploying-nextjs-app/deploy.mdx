将 Next.js 部署到生产环境的最简单方法是使用由 Next.js 的创建者开发的平台 **<PlatformLink />**。

<PlatformText/> 是一个集成全局 CDN 的平台，支持静态、 JAMstack 部署和无服务器功能。我们相信 <PlatformLink /> 是部署 Next.js 应用程序的最佳位置。你可以开始_免费_使用它 - 不需要信用卡。

### 创建一个 <PlatformText /> 账户

首先，转到 <PlatformLink href="/signup" /> 创建一个 <PlatformText /> 账户。选择 **Continue with GitHub** 并完成注册流程。

### 导入你的 `nextjs-blog` 项目

只要你登录了，你就可以在 <PlatformText /> 上**导入**你的 `nextjs-blog` 存储库。你可以从这里开始: <PlatformLink href="/import/git" />。

- 你需要**安装 <PlatformText /> for GitHub**。你可以授权它访问你的**所有仓库**。
- 一旦安装完 <PlatformText/>，导入`nextjs-blog`。

你可以对下列设置使用_默认值_ — 无需做任何修改。<PlatformText /> 自动检测到你有一个 Next.js 应用程序，并为你选择最佳的构建设置。

- Project Name
- Root Directory
- Build Command
- Output Directory
- Development Command

当你部署的时候，你的 Next.js 应用程序将开始构建，它将在一分钟内完成。

> **可获取的帮助**: 如果你的部署失败，你可以随时在 [GitHub Discussions](https://github.com/vercel/next.js/discussions) 上获得帮助。要了解有关部署的更多信息，请查看<Link href="/docs/[[...slug]]" as="/docs/deployment"><a>我们的文档</a></Link>。

完成后，你会得到**部署网址**。单击其中的一个 URLs ，你应该会看到 Next.js 的启动页。

恭喜你! 你刚刚将 Next.js 应用程序部署到了生产环境。在下一页中，我们将介绍 <PlatformText /> 的细节和推荐的工作流程。
