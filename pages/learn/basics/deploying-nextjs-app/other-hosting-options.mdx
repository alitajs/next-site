可以将 Next.js 部署到任何支持 Node.js 的托管服务器。

如果你已经遵循了目前为止的说明，那么你的 `package.json` 应该有 `build` 和 `start` 脚本:

```json
{
  "scripts": {
    "dev": "next",
    "build": "next build",
    "start": "next start"
  }
}
```

在你自己的托管服务器上，运行一次` build` 脚本，该脚本在 `.next` 文件夹中构建生产应用程序。

```shell
npm run build
```

构建完成后，`start` 脚本启动一个支持混合页面的 Node.js 服务器，它同时提供静态生成和服务端渲染的页面。该服务器也支持 API Routes。

```shell
npm run start
```

> **提示**: 你可以在 `package.json` 中自定义 `start` 脚本以接受`PORT`参数，方法是将其更新为: `"start": "next start -p $PORT"`。

就是这样! 如果你对部署Next.js有疑问，你可以在 [GitHub Discussions](https://github.com/vercel/next.js/discussions) 上询问我们的团队。
