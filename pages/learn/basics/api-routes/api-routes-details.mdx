### 不要从 `getStaticProps` 或者 `getStaticPaths` 获取 API Routes

你**不应该**从`getStaticProps`或`getStaticPaths` 获取 API Routes。相反，直接在 `getStaticProps` 或 `getStaticPaths` (或调用辅助函数)中编写服务端代码。

原因如下:`getStaticProps` 和 `getStaticPaths` 仅在服务端运行。它永远不会在客户端运行。它甚至不会包含在浏览器的 JS 包中。这意味着你可以编写直接数据库查询之类的代码，而不必将它们发送到浏览器。

### 一个好的用例: 处理表单输入

处理表单输入是一个很好的 API Routes 用例。例如，你可以在你的页面上创建一个表单，并让它向你的 API Routes 发送一个 `POST` 请求。然后，你可以编写代码直接将其保存到数据库中。API Route 代码不会是客户端包的一部分，因此你可以安全地编写服务端代码。

```js
export default (req, res) => {
  const email = req.body.email
  // Then save email to your database, etc...
}
```

### 预览模式

当你的网页从一个 headless CMS 获取数据时，静态生成是有用的。然而，当你在你的 headless CMS 上写草稿，并且想要在你的页面上立即**预览**草稿时，这并不理想。你希望 Next.js 在**请求时**而不是在构建时呈现这些页面，并获取草稿内容而不是发布的内容。你希望 Next.js 仅在这个特定情况下绕过静态生成。

Next.js 有一个特性叫做**预览模式**解决了这个问题，它利用了 API Routes。要了解更多信息，请查看我们的<Link href="/docs/[[...slug]]" as="/docs/advanced-features/preview-mode"><a>预览模式</a></Link>文档。
