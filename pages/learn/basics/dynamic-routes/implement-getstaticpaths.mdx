首先，让我们操作这些文件:

- 在 `pages/posts` 目录中创建一个名为 **`[id].js`** 的文件。
- 而且，**删除** `pages/posts` 目录中的 **`first-post.js`** 文件 — 我们将不再使用它。

然后，将以下内容添加到 `pages/posts/[id].js` 。我们稍后会填写 `...` 部分。

```jsx
import Layout from '../../components/layout'

export default function Post() {
  return <Layout>...</Layout>
}
```

然后，打开 `lib/posts.js` 并添加这个函数。这将返回 `posts` 目录中的文件名列表(排除 `.md`):

```jsx
export function getAllPostIds() {
  const fileNames = fs.readdirSync(postsDirectory)

  // Returns an array that looks like this:
  // [
  //   {
  //     params: {
  //       id: 'ssg-ssr'
  //     }
  //   },
  //   {
  //     params: {
  //       id: 'pre-rendering'
  //     }
  //   }
  // ]
  return fileNames.map(fileName => {
    return {
      params: {
        id: fileName.replace(/\.md$/, '')
      }
    }
  })
}
```

**重要的**: 返回的列表不仅仅是字符串数组 — 它**必须**是一个对象数组，看起来像上面的注释。每个对象必须有 `params` 属性，并包含一个带有 `id` 属性的对象（因为我们正在文件名中使用`[id]`)。否则，`getStaticPaths` 将失败。

最后，在 `pages/posts/[id].js` 中，我们将导入这个函数:

```js
import { getAllPostIds } from '../../lib/posts'
```

然后在 `getStaticPaths` 调用这个函数:

```jsx
export async function getStaticPaths() {
  const paths = getAllPostIds()
  return {
    paths,
    fallback: false
  }
}
```

- 返回对象的 `paths` 属性必须是 `id` 的可能值数组。这正是`getAllPostIds()`返回的内容。
- 现在忽略 `fallback: false`，我们稍后会解释。
