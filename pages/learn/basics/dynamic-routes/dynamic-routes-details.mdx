### 获取外部 API 或查询数据库

与 `getStaticProps` 类似，`getStaticPaths` 可以从任何数据源获取数据。在我们的示例中，`getAllPostIds` 的数据(由 `getStaticPaths` 使用)可以从外部API获取:

```js
export async function getAllPostIds() {
  // Instead of the file system,
  // fetch post data from an external API endpoint
  const res = await fetch('..')
  const posts = await res.json()
  return posts.map(post => {
    return {
      params: {
        id: post.id
      }
    }
  })
}
```

### 开发环境 v.s. 生产环境

- 在 **development** 中 (`npm run dev` or `yarn dev`), `getStaticProps` 在_每个请求上_运行。
- 在 **production** 中, `getStaticProps` 在_构建时_运行。

### Fallback

还记得吗，我们从 `getStaticPaths` 返回了 `fallback: false` ，这是什么意思？

如果 `fallback` 是 `false`，那么 `getStaticPaths` 没有返回的任何路径将导致一个**404 page**。

如果 `fallback` 是 `true`，那么 `getStaticPaths` 的行为就会改变:

- 在构建时间从 `getStaticPaths` 返回的路径将被渲染到 HTML。
- 在构建时未生成的路径**不会**导致404页。相反，Next.js 将在第一个请求中提供该页面的 “fallback” 版本。
- 在后台，Next.js 将静态生成请求的路径。生成的页面将服务于对同一路径的后续请求，就像构建时预渲染的其他页面一样。

这超出了我们的课程的范围，但你可以在<Link href="/docs/[[...slug]]" as="/docs/basic-features/data-fetching#fallback-pages"><a>我们的文档</a></Link>学到更多关于`fallback: true`的知识。

### 全面控制路由

可以通过在括号内添加三个点 (`...`) 来扩展动态路由以捕捉所有路径。例如:

- `pages/posts/[...id].js` 匹配 `/posts/a`, 但也匹配 `/posts/a/b`, `/posts/a/b/c` 等。

如果你这样做，在 `getStaticPaths` 中，你必须返回一个数组作为 `id` 键的值，如下所示:

```js
return [
  {
    params: {
      // Statically Generates /posts/a/b/c
      id: ['a', 'b', 'c']
    }
  }
  //...
]
```

然后在 `getStaticProps` 中， `params.id` 将是一个数组:

```js
export async function getStaticProps({ params }) {
  // params.id will be like ['a', 'b', 'c']
}
```

### 404 页面

若要创建自定义404页，请创建 `pages/404.js`。此文件在编译时静态生成。

```jsx
// pages/404.js
export default function Custom404() {
  return <h1>404 - Page Not Found</h1>
}
```
