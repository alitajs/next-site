要使用服务器端呈现，你需要从页面中导出 `getServerSideProps` 而不是导出 `getStaticProps`。

### 使用 `getServerSideProps`

下面是 `getServerSideProps` 的演示代码。对于我们的博客示例来说，它是不必要的，所以我们不会实现它。

```jsx
export async function getServerSideProps(context) {
  return {
    props: {
      // props for your component
    }
  }
}
```

因为 `getServerSideProps` 是在请求时调用的，所以它的参数(`context`)包含请求特定的参数。你可以在<Link href="/docs/[[...slug]]" as="/docs/basic-features/data-fetching#getserversideprops-server-side-rendering"><a>我们的文档</a></Link>中了解更多信息。

只有当你需要预呈现一个必须在请求时获取数据的页面时，你才应该使用`getServerSideProps`。到第一个字节的时间(TTFB)会比 `getStaticProps` 慢，因为服务器必须计算每个请求的结果，而且如果没有额外的配置，结果不能被CDN缓存。

### 客户端渲染

如果你**不**需要预渲染的数据，你也可以使用以下策略(称为**客户端渲染**):

- 静态生成不需要外部数据的页面部分(预渲染)。
- 当页面加载时，使用 JavaScript 从客户端获取外部数据并填充其余部分。

例如，这种方法适用于用户仪表板页面。因为仪表板是一个私有的、用户特定的页面，所以 SEO 不相关，而且页面不需要预先渲染。数据经常更新，还需要请求时获取数据。

### SWR

Next.js 的团队创建了一个用于数据获取的 React hook，称为[**SWR**](https://swr.now.sh/)。我们强烈建议你在客户端获取数据是使用它。它处理缓存、重新验证、焦点跟踪、间隔重构等等。在这里我们不讨论细节，但这里有一个示例用法:

```jsx
import useSWR from 'swr'

function Profile() {
  const { data, error } = useSWR('/api/user', fetch)

  if (error) return <div>failed to load</div>
  if (!data) return <div>loading...</div>
  return <div>hello {data.name}!</div>
}
```

[了解更多 SWR 信息](https://swr.now.sh/).
